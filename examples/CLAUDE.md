# CLAUDE.md

本文件为 Claude Code 用户级全局配置，适用于所有项目。

---

## 核心理念

你是 Claude Code。复杂任务委派给专业代理和技能。

**关键原则：**
1. **代理优先** — 复杂工作委派给专业代理
2. **并行执行** — 尽可能使用 Task 工具并行启动多个代理
3. **先规划后执行** — 复杂操作使用 Plan Mode
4. **测试驱动** — 先写测试再实现
5. **安全优先** — 安全问题绝不妥协

## 个人偏好

### 代码风格
- 代码、注释、文档中不使用 emoji
- 优先不可变 — 不要修改对象或数组
- 多个小文件优于少量大文件
- 典型 200-400 行，单文件最多 800 行

### Git
- 约定式提交：`feat:`、`fix:`、`refactor:`、`docs:`、`test:`
- 提交前本地测试通过
- 小而聚焦的提交

### 测试
- TDD：先写测试
- 最低 80% 覆盖率
- 关键流程需要单元 + 集成 + E2E

### 知识沉淀
- 存在command，learned。可以提取当前上下文有用的信息进行知识沉淀。
- 个人调试笔记、偏好、临时上下文 → auto memory
- 团队/项目知识（架构决策、API 变更、实现手册）→ 遵循项目已有文档结构
- 如果当前任务已产出相关文档/注释/示例，不重复记录
