### 4.2 关键 LOG 短句释义

| LOG 短句 | 代表的关键信息 |
|----------|----------------|
| `[入口日志短句]` | 代表消息/信号已经进入目标模块；如果缺失，优先检查上游是否触发、日志过滤条件是否正确。 |
| `[状态更新日志短句]` | 代表关键状态已经被写入或刷新；重点确认状态值、来源和是否被后续覆盖。 |
| `[业务决策日志短句]` | 代表条件判断已经执行；重点确认命中的分支、关键条件值和被拒绝的原因。 |
| `[对外动作日志短句]` | 代表已经向页面、协议、事件或回调发出动作；如果后续无效果，继续查下游接收与结果确认。 |
| `[结果确认日志短句]` | 代表动作已经被确认、完成或失败；重点确认成功标志、错误码、超时和最终状态。 |
